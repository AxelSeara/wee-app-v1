# Security Policy

## Supported Versions

We currently support the latest `main` branch for security fixes.

| Version | Supported |
| ------- | --------- |
| main    | yes       |
| older tags/releases | no |

## Reporting a Vulnerability

Please do **not** post security vulnerabilities in public issues.

Preferred channels:

1. GitHub Security Advisory (private report), if enabled.
2. If advisory is unavailable, open an issue titled `[security] private report requested` with no exploit details.

When reporting, include:

- Affected area/files
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

We will acknowledge reports as soon as possible and aim to provide a first triage within 72 hours.
